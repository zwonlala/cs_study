# 쓰레드 (Thread)

쓰레드와 프로세스와 유사한 게 있음

쓰레드 : 프로그램 내부의 흐름, 맥

- 다중 쓰레드 
한 프로그램에 2개 이상의 맥
맥이 빠른 시간 간격으로 스위칭 됨
=> 여러개의 맥이 동시에 실해되는 것 처럼 보임

콘커런트하다(동시하다) <-> 시뮬테이너스 (이건 실제로 씨피유가 2개 이상이어야 가능!)

ex web browser
화면 출력하는 쓰레드
데이터 읽어오는 쓰레드

ex word processor
화면 출력하는 쓰레드 + 키보드 입력 받는 쓰레드 + 철자/문법 오류 확인 쓰레드


요즘 오에스는 다중 쓰레드를 지원

하나의 프로세스 안에 여러 쓰레드가 있음

p1 프로새스 실행할때, 첫번째 쓰레드 실행하다가, 두번째 쓰레드 실행함

쓰레드는 서로 메모리 공유
(코드, 데이터는 공유 & 스택은 공유하지 않음)
p1 프로세스가 가지고 있는 자원 공유(파일, 아이오)
비공유 : 개별적인 PC, sp 값, 레지스터 값

오에스를 공부할때, 이전에는 프로세스 단위로 스위칭 되지만,
요즘 오에스는 쓰레드 단위로 콘텍스트 스위칭 일어남
